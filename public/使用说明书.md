## 1. ScopeGuard 是什么？

ScopeGuard 是一个帮你 **追踪"合同外需求"** 的小工具。

### 场景举例

> 你接了一个品牌设计项目，合同上写的是：Logo × 3 稿 + 名片 + 信封信纸。  
> 做了一半，客户说"顺便把头图也做了吧"，你顺手做了。  
> 客户又说"小红书封面也来几张"，你又做了。  
> **最后交付物是合同的 3 倍，钱没多一分。**

ScopeGuard 就是来解决这个问题的。它让你：

1. **建项目时** —— 录入合同上的交付物清单
2. **客户加需求时** —— 花 10 秒标记一下是"合同内"还是"超范围"
3. **项目结束时** —— 一眼看出：合同约定 X 项，实际做了 Y 项，白干了 Z 项
4. **（Pro 功能）** —— 一键把超范围的需求变成报价单发给客户

## 5. 怎么标记客户加的需求？

**这是 ScopeGuard 最核心的功能。** 也是你保护自己的关键操作。

### 什么时候需要标记？

> 客户在微信上给你发了一条消息：
> *"顺便帮我再做一个公众号头图吧，跟品牌风格统一一下就行。"*

这时候，你打开项目详情页面，看到右下角有一个 **绿色的 + 按钮**：

```
                           ┌─────┐
                           │  +  │
                           └─────┘
```

点击它，弹出下面的窗口：

```
┌────────────────────────────────────┐
│       标记新增需求                  │
│                                    │
│  需求描述                          │
│  ┌──────────────────────────┐     │
│  │ 客户要求增加公众号头图    │     │
│  └──────────────────────────┘     │
│                                    │
│  估算工时（小时）                  │
│  ┌──────────────────────────┐     │
│  │  3                      │     │
│  └──────────────────────────┘     │
│                                    │
│  需求类型                          │
│                                    │
│  ┌──────────────┐ ┌────────────┐  │
│  │ 🔴超范围新增  │ │ 合同内修订 │  │
│  └──────────────┘ └────────────┘  │
│                                    │
│   ┌──────────────────────────┐    │
│   │        保存标记           │    │
│   └──────────────────────────┘    │
└────────────────────────────────────┘
```

### 填什么？

| 字段 | 说明 |
|------|------|
| **需求描述** | 客户要你做什么？写清楚（以后要拿这个跟客户对账的） |
| **估算工时** | 这个活儿大概要花你几个小时？随便估一个数就行 |
| **需求类型** | **这是关键！** 选"超范围新增"还是"合同内修订" |

### 怎么选"超范围"还是"合同内"？

对照你之前录入的**交付物清单**来判断：

| 如果是这种情况 | 选 |
|--------------|----|
| 交付物清单上有这项，客户只是要求修改 | **合同内修订** |
| 交付物清单上**没有**这项，客户额外加的 | **超范围新增** ⬅️ 选这个！ |
| 不确定是不是合同内的 | **超范围新增**（宁可多标记，不要漏掉） |

> 💡 **简单判断法：** 如果这个需求在合同里没写，那就是超范围。**先标上再说。**

### 保存之后会怎样？

标记保存后：
1. 右上角的**偏差看板卡片**会实时更新
2. 下方的**需求时间线**会多一条记录
3. 如果积累了多个超范围需求，Pro 用户可以生成报价单

## 6. 怎么看项目有没有偏离合同？

在项目详情页下方，有 4 张**数据卡片**：

```
┌──────────┐ ┌──────────┐
│ 合同进度  │ │超范围新增 │
│  3/4项   │ │  2项     │
│ 完成 75% │ │进行中1项  │
└──────────┘ └──────────┘
┌──────────┐ ┌──────────┐
│ 额外工时  │ │时薪损失   │
│  6.0h    │ │ ¥1,200   │
│≈0.8个工作日│ │时薪¥200/h│
└──────────┘ └──────────┘
```

### 每张卡片什么意思？

| 卡片 | 看什么 | 例子 |
|------|--------|------|
| **合同进度** | 合同内的活儿做了多少 | 3/4 项 = 做了 3 项，还有 1 项没做 |
| **超范围新增** | 客户额外加了多少需求 | 2 项 = 客户已经加了 2 个"顺便" |
| **额外工时** | 白干了多少小时 | 6h = 多干了差不多一个工作日 |
| **时薪损失** | 折算成钱亏了多少 | ¥1,200 = 按你的时薪算损失 |

> ⚠️ **这个看板是你跟客户谈判的底气。** 下次客户再说"小需求而已"，你就把屏幕给他看："这几个小需求加起来已经 ¥1,200 了。"

## 10. 常见问题

### Q：免费版只有 2 个项目，用完了怎么办？

你可以把已完成的项目**归档**，释放额度。或者升级到 Pro 版，不限项目数。

### Q：我标错了"超范围"，能改吗？

可以。到"需求记录"页面，找到那条记录，点击编辑 ✎ 就可以改描述和工时。如果要切换"超范围/合同内"，目前需要删除后重新标记。

### Q：客户不承认是超范围怎么办？

这就是 ScopeGuard 的作用——**从项目一开始就把基线定好**。每次加需求时标记"超范围"，到沟通的时候你有完整的时间线和数据。你可以说："合同上写的是这 4 项，这 2 项是后来您加的，我们看一下怎么处理。"

### Q：时薪我填的是人民币还是美元？

都可以。系统只按数字算账，你填什么货币单位就是什么单位。中国用户填人民币即可。

### Q：数据安全吗？会被别人看到吗？

ScopeGuard 是**单用户工具**，你的项目只有你自己能看到。分享报价单给客户时，系统生成一个**随机链接**，只有有这个链接的人才能看到。

### Q：我想导出数据怎么办？

Pro 版支持报价单分享链接。详细的数据导出功能会在后续版本中提供。

### Q：在手机上能用吗？

**当然可以。** ScopeGuard 是专门为手机优化的。所有页面在手机上都能流畅操作，尤其是"标记超范围需求"这个动作，在手机上 10 秒就能完成——客户在微信说了什么，你直接复制粘贴进来就行。

## 最后的话

ScopeGuard 的理念很简单：**不让你白干。**

在自由职业的世界里，"范围蔓延"不是一个专业术语，它是**你的钱**，**你的时间**，**你的生活**。

从现在开始，每个项目用 ScopeGuard 记录一下。不需要多复杂——花 3 分钟建项目，每次客户加需求时花 10 秒标记。到月底，你就能清清楚楚地看到：**这个月我到底干了多少合同外的活儿，值多少钱。**

> 📌 **记住三个动作：**
> 1. **建项目时** → 把合同交付物抄进去
> 2. **客户加需求时** → 点 + 号标记"超范围"
> 3. **项目结束时** → 看看偏差看板，该收的钱收回来
