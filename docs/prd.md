## 1. 产品概述

| 项目 | 内容 |
|------|------|
| 产品名 | ScopeGuard（范围蔓延守卫） |
| 一句话 | 帮助自由职业者量化合同外需求并自动生成变更报价 |
| 目标用户 | 华语自由职业者（独立设计师、开发者、文案、视频剪辑师） |
| 技术栈 | Next.js + Supabase |
| 变现模式 | Freemium（2 个免费项目）→ ¥39/月 Pro |
| 核心理念 | 三个按钮，不是三百个 |

**痛点**：自由职业者缺乏工具追踪"合同内交付物 vs 客户额外加的需求"，导致范围蔓延悄悄稀释时薪。平均每月损失 $1,000+。

## 2. MVP 三大功能拆解

### 2.1 范围边界设定器（免费）

**用户故事**

> 作为自由职业者，我想在项目开始时录入合同约定的交付物清单，以便建立"范围基线"，后续能清晰区分哪些需求是合同内的、哪些是额外加的。

**功能流程**

```
创建项目 → 填写项目名称/客户名称 → 录入交付物列表（每项含名称+备注） → 设定项目费率 → 完成基线设定
```

- 交付物列表支持纯文本输入，每行一条
- 每个交付物可标记三种状态：未开始 / 进行中 / 已完成
- 支持编辑和删除已录入的交付物
- 整个操作不超过 3 分钟

**涉及页面**
- `/projects/new` — 新建项目页
- `/projects/[id]` — 项目详情页（交付物列表）

---

### 2.2 超范围标记 + 偏差看板（免费）

**用户故事**

> 作为自由职业者，当客户在微信里提出"顺便再加一个小功能"时，我想花 10 秒标记该需求为"超范围"，系统会自动告诉我：合同内完成度还剩多少、额外增加了多少工作量。

**功能流程**

```
收到客户新增需求 → 在项目看板点击"标记超范围" → 输入需求描述/估算工时 → 自动计入偏差统计 → 看板实时更新
```

- 看板展示核心指标：
  - 原定交付物 X 项，已完成 Y 项（进度 Z%）
  - 超范围新增 A 项，估算额外工时 B 小时
  - 时间线对比：原定范围 vs 实际交付
- 支持标记"合同内修订"（不纳入超范围统计）
- 支持撤回/编辑标记

**涉及页面**
- `/projects/[id]` — 项目详情页（偏差看板主体）
- `/projects/[id]/requests` — 需求列表页（所有标记记录）

---

### 2.3 一键变更报价单（Pro）

**用户故事**

> 作为想要收取超范围工作报酬的自由职业者，我想一键将所有超范围需求汇总成一份专业的变更报价单，发送给客户确认成交。

**功能流程**

```
查看超范围需求列表 → 点击"生成变更报价单" → 系统自动汇总所有超范围条目 → 填入报价单价 → 生成报价单 → 复制分享链接/导出 PDF
```

- 报价单包含：需求描述、估算工时、建议报价、合计金额
- 免费版可预览报价单内容，但**不可发送/分享**
- Pro 版一键生成，支持链接分享和 PDF 导出
- 客户可在线确认或协商

**涉及页面**
- `/projects/[id]/change-order` — 变更报价单页（预览/编辑/分享）

## 5. 非功能需求

- **响应式 / 移动端优先**：核心流程（标记超范围、看板）在手机上顺畅操作
- **中文界面**：UI 文案默认简体中文，不提供英文版
- **3 分钟上手**：新用户创建项目+录入交付物清单总时长 ≤ 3 分钟
- **单用户工具**：无团队协作、无多角色、无权限管理
- **SSR + SPA**：仪表盘看板需实时更新，页面切换无需整页刷新
- **离线友好**：标记需求和查看看板应支持弱网环境

---

## 6. 验收标准

1. 用户可创建项目并录入交付物列表，每项可标记状态
2. 用户可标记需求为"合同内修订"或"超范围新增"
3. 偏差看板实时展示：合同内进度、超范围数量/工时、时间线对比
4. Pro 用户可一键生成变更报价单并分享链接
5. 免费用户可预览报价单内容，不可点击"发送/分享"
6. 所有页面在手机端和桌面端均可正常使用
7. 新用户从注册到完成首个项目设定 ≤ 3 分钟
8. 免费版限制最多 2 个活跃项目
