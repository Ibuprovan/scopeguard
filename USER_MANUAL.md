## 一、ScopeGuard 是什么

**ScopeGuard（范围蔓延守卫）** 是一个帮你追踪"客户额外加的需求"并自动生成变更报价单的小工具。

**典型使用场景**：

你接了一个设计项目，合同里写了 5 个交付物。但客户中途说"顺便再加一个 Logo"、"再多改两稿"、"再加个banner"——这些就是"范围蔓延"。ScopeGuard 帮你：

1. 把合同里约定的交付物记下来（建立"基线"）
2. 客户每次加需求时，10 秒钟标记一下"超范围"还是"合同内"
3. 实时看到：你已经被白嫖了多少工时、损失了多少钱
4. 一键生成专业的变更报价单，发给客户收钱

**一句话**：让"再加一个需求"从口头免费，变成可量化、可收费。

## 二、第一次使用前的准备工作

整个过程大约需要 **20-30 分钟**，主要时间花在注册 Supabase 上。只要做一次，以后用就只需要双击一下。

需要做 5 件事：

1. 安装 Node.js（一次性，5 分钟）
2. 下载本项目（一次性，1 分钟）
3. 注册并配置 Supabase（一次性，15 分钟）
4. 一键启动（每次用都做，10 秒）
5. 注册你的账号（一次性，2 分钟）

---

### 第 1 步：安装 Node.js

Node.js 是运行本项目的"引擎"，必须装。

#### 1.1 下载

1. 打开浏览器，访问 **https://nodejs.org**
2. 你会看到两个绿色下载按钮，点 **左边那个写着 "LTS" 的**（Recommended For Most Users）
3. 等待下载完成

> **Mac 用户**：也可以用终端执行 `brew install node` 安装（如果你装过 Homebrew）。

#### 1.2 安装

**Windows 用户**：

1. 双击下载的 `.msi` 文件
2. 一路点 "Next"（下一步）
3. 看到 "Install" 点 "Install"（如果弹窗问"是否允许更改"，点"是"）
4. 看到 "Finish" 点 "Finish"（完成）

**Mac 用户**：

1. 双击下载的 `.pkg` 文件
2. 一路点"继续"，最后点"安装"
3. 输入你的电脑密码确认

#### 1.3 验证安装是否成功

**Windows 用户**：

1. 按键盘上的 `Win + R` 键（Win 键是左下角带 Windows 图标的那个键）
2. 在弹出的"运行"窗口里输入 `cmd`，按回车
3. 在弹出的黑色窗口里输入：`node -v`
4. 按回车，如果看到类似 `v20.11.0` 这样的版本号，说明安装成功

**Mac 用户**：

1. 按 `Command + 空格`，输入 `Terminal`，按回车打开终端
2. 输入 `node -v`，按回车
3. 看到类似 `v20.11.0` 即安装成功

> 如果提示"node 不是内部或外部命令"，说明安装没成功，请重启电脑后再试一次；还不行就重新安装。

---

### 第 2 步：下载本项目

#### 方式 A：下载 ZIP 压缩包（推荐小白用）

1. 打开浏览器，访问项目地址：**https://github.com/Ibuprovan/scopeguard**
2. 在页面右侧找到绿色的 **`<> Code`** 按钮，点击它
3. 在弹出菜单最下方点 **`Download ZIP`**
4. 等待下载完成
5. 把 ZIP 文件解压到一个**容易找到的位置**，建议：
   - Windows：解压到 `C:\Users\你的用户名\Desktop\` （桌面）
   - Mac：解压到 `~/Desktop/` （桌面）

#### 方式 B：用 git 克隆（如果你会用 git）

```bash
git clone https://github.com/Ibuprovan/scopeguard.git
cd scopeguard
```

解压后你会得到一个名为 `scopeguard` 或 `scopeguard-main` 的文件夹。**记住这个文件夹的位置**，后面要用。

---

### 第 3 步：注册并配置 Supabase

Supabase 是这个项目用的"云端数据库"。ScopeGuard 本身不存数据，数据都存在你的 Supabase 账户里。

> **为什么要自己注册？** 因为数据是你自己的（项目、客户、报价单），不应该交给项目作者。这样即使作者不维护了，你的数据依然在你的 Supabase 账户里，完全归你所有。

#### 3.1 注册 Supabase 账号

1. 打开 **https://supabase.com**
2. 右上角点 **`Sign in`** → **`Sign up`**
3. 输入邮箱和密码注册（推荐用 GitHub 账号登录，更快）
4. 验证邮箱（去邮箱点确认链接）

#### 3.2 创建一个 Supabase 项目

1. 登录后，点 **`New project`**（新建项目）
2. 填写项目信息：
   - **Name**：填 `scopeguard`（随便起，方便辨认）
   - **Database Password**：点 **`Generate a password`** 自动生成，然后**把密码复制保存到记事本**（虽然这个项目用不到，但万一以后要管理数据库会用到）
   - **Region**：选离你最近的，国内用户选 **`Southeast Asia (Singapore)`** 或 **`Northeast Asia (Tokyo)`**
3. 点 **`Create new project`**
4. 等待约 1-2 分钟，页面跳转后即创建完成

#### 3.3 拿到两个关键信息

1. 在 Supabase 项目页面，左侧菜单找到 **`Project Settings`**（齿轮图标，通常在最下方）
2. 点 **`API`**
3. 你会看到页面上有这几个值，**把它们记下来**：

   - **Project URL**：形如 `https://xxxxxxxxxxxxx.supabase.co`
   - **Project API Keys** → **`anon` `public`**：一长串 `eyJhbGciOi...` 开头的字符串

> 注意：是 `anon` `public` 那个 key，**不是** `service_role` 那个。`service_role` 是管理员密钥，绝对不要泄露。

#### 3.4 配置 .env.local 文件

1. 用记事本（Mac 用文本编辑）打开你刚才解压的 `scopeguard` 文件夹
2. 找到名为 **`.env.local.example`** 的文件
3. 把它**复制一份**，把副本重命名为 **`.env.local`**（注意：去掉 `.example` 后缀）

   > **Windows 小提示**：如果改完名字又变回 `.env.local.example.txt`，是因为系统隐藏了扩展名。在文件夹顶部菜单 →"查看"→勾选"文件扩展名"，再重命名一次。

4. 用记事本打开 `.env.local`，你会看到：

   ```env
   NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
   NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
   ```

5. 把等号右边的内容替换成你刚才记下的真实值：

   ```env
   NEXT_PUBLIC_SUPABASE_URL=https://xxxxxxxxxxxxx.supabase.co
   NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
   ```

6. 保存文件（Ctrl + S），关闭记事本

> **重要**：URL 和 key 之间不要有空格，也不要加引号。

#### 3.5 执行数据库初始化 SQL

这一步是给数据库"建表"，告诉 Supabase 数据要怎么存。

1. 回到 Supabase 项目页面
2. 左侧菜单点 **`SQL Editor`**（图标像个数据库）
3. 点右上角 **`+ New Query`**（新建查询）
4. 打开你电脑上的 `scopeguard` 文件夹，进入 **`supabase`** 子文件夹
5. 用记事本打开 **`init.sql`** 文件
6. 按 `Ctrl + A` 全选，按 `Ctrl + C` 复制
7. 回到 Supabase 的 SQL Editor，把内容粘贴到编辑区
8. 点右下角绿色的 **`Run`** 按钮（或按 `Ctrl + Enter`）
9. 等待几秒，看到下方输出 **`Success. No rows returned`** 即表示成功

> 如果看到红色的错误提示，请检查：
> - 是不是之前已经执行过一次了？（重复执行会报"already exists"）
> - 复制粘贴时有没有遗漏内容？

到这里，准备工作全部完成！接下来就是见证奇迹的时刻。

---

### 第 4 步：一键启动

#### Windows 用户

1. 打开你解压好的 `scopeguard` 文件夹
2. 找到名为 **`start.bat`** 的文件
3. **双击它**
4. 会弹出一个黑色窗口，显示检查进度：
   - `[√] 已检测到 Node.js`
   - `[√] 配置文件 .env.local 已就绪`
   - `[*] 首次运行，正在安装依赖包...` （这一步可能要等 1-3 分钟，**不要关闭窗口**）
   - `[√] 依赖安装完成`
   - `正在启动... 浏览器会在 5 秒后自动打开`
5. 浏览器会自动打开 `http://localhost:3000`，看到 ScopeGuard 首页即表示成功

> **首次启动会慢一些**（要下载依赖），以后每次启动只需几秒钟。
>
> **如果窗口一闪而过**：说明有错误。在文件夹空白处按住 `Shift` + 右键 → "在此处打开 PowerShell"，手动运行 `.\start.bat` 查看错误。

#### Mac / Linux 用户

1. 打开终端（`Command + 空格` 输入 `Terminal`）
2. 用 `cd` 命令进入项目文件夹，例如：

   ```bash
   cd ~/Desktop/scopeguard
   ```

3. 给脚本执行权限（只需做一次）：

   ```bash
   chmod +x start.sh
   ```

4. 运行脚本：

   ```bash
   ./start.sh
   ```

5. 浏览器会自动打开，看到首页即成功

> **启动期间不要关闭终端窗口**。关闭终端 = 停止程序。

### 第 5 步：注册你的第一个账号

1. 在浏览器打开的首页上，点 **`开始免费使用`** 按钮
2. 跳转到登录页，输入你的邮箱地址
3. 点 **`发送登录链接`**
4. 页面变成"检查你的邮箱"
5. 去你的邮箱（注意查一下**垃圾邮件**），会收到一封来自 Supabase 的邮件
6. 点邮件里的登录链接
7. 浏览器会自动跳转回 ScopeGuard，进入仪表盘
8. 搞定！你已经登录成功了

> **登录链接有时效**：一般是 1 小时内有效，过期了就重新发一次。
>
> **第一次登录会比较慢**：因为系统要在数据库里给你创建档案，可能要等 5-10 秒。

## 三、日常使用指南

### 4.1 创建第一个项目

1. 在仪表盘页面，点 **`新建项目`** 按钮（首次使用时会显示"创建第一个项目"大按钮）
2. 填写项目信息：

   | 字段 | 说明 | 例子 |
   |------|------|------|
   | **项目名称** | 给这个项目起个名字 | `品牌VIS设计` |
   | **客户名称** | 客户公司或个人名 | `某科技有限公司` |
   | **时薪 (¥/h)** | 你的时薪，用于后续计算损失和报价 | `200` |

3. 在 **`交付物清单`** 文本框里，**每行写一个**合同约定的交付物：

   ```
   Logo设计（3稿）
   名片设计（正反面）
   信封设计
   信纸设计
   ```

4. 下方会实时显示"已录入 4 个交付物"
5. 点 **`创建项目`** 按钮
6. 跳转到项目详情页，项目创建完成

> **小贴士**：交付物清单是"范围基线"，越完整越好。所有没出现在这个清单里的需求，都可能是"超范围"。

---

### 4.2 录入交付物清单

如果你创建项目时漏了录入交付物，或者想事后修改，可以在项目详情页操作：

#### 查看交付物

进入项目详情页后，向下滚动看到 **`交付物清单`** 区域，列出你录入的所有交付物。

#### 切换交付物状态

每个交付物左侧有一个圆形按钮，点击它会循环切换状态：

- **未开始**（灰色）→ 点击 → **进行中**（蓝色）
- **进行中** → 点击 → **已完成**（绿色带对勾）
- **已完成** → 点击 → 回到 **未开始**

#### 编辑交付物名称

1. 鼠标悬停在某个交付物上
2. 右侧会出现铅笔图标
3. 点击铅笔图标，名称变成可编辑的输入框
4. 修改后按回车，或点击空白处保存

#### 删除交付物

1. 鼠标悬停在某个交付物上
2. 右侧会出现垃圾桶图标
3. 点击即可删除（**注意：无二次确认，点错了直接没了**）

#### 添加新交付物

> 注意：当前版本在项目创建后，**没有"添加新交付物"的入口**。请创建项目时一次录入完整。如果确实需要加，可以新建一个项目。

---

### 4.3 标记客户的新需求

这是 ScopeGuard 的核心功能——客户每次加需求，10 秒钟标记一下。

#### 标记需求

1. 在项目详情页，**右下角有一个圆形的 `+` 按钮**
2. 点击它，弹出"标记新增需求"窗口
3. 填写：

   | 字段 | 说明 |
   |------|------|
   | **需求描述** | 客户具体要什么（写清楚点，方便后面算钱） |
   | **估算工时（小时）** | 大概要花多少小时做完，填数字（可填小数，如 `4.5`） |
   | **需求类型** | 二选一（见下方） |

4. **需求类型**选一个：
   - **`超范围新增`**（红色高亮）：合同里没有的，客户额外加的，要收费
   - **`合同内修订`**（绿色高亮）：合同范围内的小调整，不收费

5. 点 **`保存标记`**
6. 需求会出现在上方的"需求时间线"里，偏差看板也会立即更新

#### 查看所有需求

1. 在项目详情页向下滚动到 **`需求时间线`** 区域
2. 默认显示最近 5 条
3. 点右上角 **`查看全部`**，进入完整的需求列表页

#### 在需求列表页操作

- **筛选**：点顶部的 **`全部`** / **`超范围`** / **`合同内`** 标签切换查看
- **编辑**：点需求右侧的铅笔图标，修改后点 **`保存`**
- **删除**：点需求右侧的垃圾桶图标（**无二次确认**）

---

### 4.4 查看偏差看板

在项目详情页，交付物清单下方就是 **`偏差看板`**。这是 ScopeGuard 的"灵魂"。

看板包含 4 个核心统计卡片 + 1 个时间线对比：

#### 4 个统计卡片

| 卡片 | 含义 | 怎么看 |
|------|------|--------|
| **合同进度** | 合同内交付物完成情况 | `3/5 完成 60%` 表示 5 个交付物完成了 3 个 |
| **超范围新增** | 客户额外加了多少需求 | `4项` 表示已经累计 4 个超范围需求 |
| **额外工时** | 超范围需求累计的工时 | `12h` 意味着客户白嫖了你 12 小时 |
| **时薪损失预估** | 折算成钱的损失 | `¥2400` 意味着你已经损失了 2400 块（按你的时薪算） |

#### 时间线对比

| 进度条 | 含义 |
|--------|------|
| **范围蔓延** | 原始交付物数 → 实际总数（含超范围），增长百分比 |
| **工时偏差** | 原始预估工时 → 实际工时（含超范围） |
| **预算偏差** | 原始预算 → 实际应得金额（含超范围部分） |

> **算法说明**（MVP 简化版）：每个原始交付物默认按 8 小时算，原始预算 = 原始工时 × 你的时薪。这是简化算法，未来版本会支持每个交付物单独设置工时。

#### 怎么用这些数据

- **跟客户沟通时**：截图偏差看板，让客户直观看到"我已经多做了多少"
- **决策时**：损失超过 1 天工时（8 小时）就该考虑发报价单了
- **复盘时**：项目结束后看哪个客户最喜欢加需求，下次报价留余地

---

### 4.5 生成并分享变更报价单

当超范围需求积累到一定程度，你可以一键生成专业的变更报价单发给客户。

#### 生成报价单

1. 在项目详情页（必须有至少 1 个"超范围新增"需求），向下滚动到底部
2. 看到 **`生成变更报价单`** 按钮（绿色全宽按钮）
3. 点击它，跳转到变更报价单页
4. 页面会自动汇总所有"超范围新增"的需求，按你的时薪计算每项金额
5. 查看合计金额（绿色加粗）
6. 点 **`生成正式报价单`** 按钮

#### 分享给客户

1. 生成报价单后，会出现 **`发送并复制分享链接`** 按钮
2. 点击它，链接会自动复制到你的剪贴板
3. 看到页面顶部蓝色提示 **`分享链接已复制到剪贴板`**
4. 把链接粘贴到微信、邮件、QQ 发给客户
5. 客户点开链接就能看到报价单（**客户不需要注册账号**）

#### 重新生成（有新需求时）

如果你发过报价单之后客户又加了新需求：

1. 回到变更报价单页
2. 点 **`重新生成报价单`** 按钮
3. 新需求会自动并入
4. 再点 **`复制分享链接`** 重新发给客户

#### 报价单包含什么

- 项目名称 + 客户名称
- 每条超范围需求的描述、工时、金额
- 合计金额
- 一个分享令牌（保证链接唯一不可猜测）

## 四、日常使用（下次再用）

第一次配置好之后，以后每次用都超级简单：

### 启动

**Windows**：双击 `scopeguard` 文件夹里的 **`start.bat`**

**Mac/Linux**：终端执行：

```bash
cd ~/Desktop/scopeguard
./start.sh
```

浏览器会自动打开，1-2 秒就能用。

### 使用

1. 浏览器打开 `http://localhost:3000`
2. 如果显示登录页，输入邮箱收登录链接（无需密码）
3. 进入仪表盘，正常使用

### 停止

见下一章。

---

## 五、如何停止程序

**Windows**：

- 直接关闭那个黑色的命令行窗口即可
- 或者在窗口里按 `Ctrl + C`

**Mac/Linux**：

- 在终端里按 `Ctrl + C`
- 或者直接关闭终端窗口

> **数据不会丢失**：你录入的所有数据都存在 Supabase 云端，跟本地程序无关。下次启动依然能看到。

## 六、如何更新到新版本

如果项目作者发布了新版本，你想升级：

### 方式 A：重新下载 ZIP（最简单）

1. 备份你的 `.env.local` 文件（**重要！** 里面有你的 Supabase 配置）
2. 删掉旧的 `scopeguard` 文件夹
3. 重新去 https://github.com/Ibuprovan/scopeguard 下载 ZIP
4. 解压，把备份的 `.env.local` 文件放回去
5. 双击 `start.bat` 启动

### 方式 B：用 git 拉取更新（如果你会用 git）

```bash
cd scopeguard
git pull
# 如果有新依赖
npm install
```

> **数据库要不要更新**：如果新版本有数据库结构变化，作者会在更新日志里说明，并附上需要执行的 SQL。一般小版本更新不需要。

## 七、常见问题 FAQ

### Q1：双击 start.bat 后窗口一闪就关了

**原因**：可能是 Node.js 没装好，或者 .env.local 配置有问题。

**解决**：

1. 按 `Win + R`，输入 `cmd`，回车
2. 用 `cd` 命令进入项目文件夹，例如：

   ```
   cd C:\Users\你的用户名\Desktop\scopeguard
   ```

3. 手动运行 `start.bat`
4. 这次窗口不会自动关闭，能看到具体错误信息
5. 截图发给懂技术的朋友求助

---

### Q2：依赖安装很慢或者失败

**原因**：默认 npm 源在国外，国内网络访问慢。

**解决**：切换到国内镜像

1. 按 `Win + R` 输入 `cmd` 打开命令行
2. 执行：

   ```
   npm config set registry https://registry.npmmirror.com
   ```

3. 删除项目里的 `node_modules` 文件夹和 `package-lock.json` 文件
4. 重新双击 `start.bat`

---

### Q3：登录链接收不到邮件

**解决**：

1. 检查**垃圾邮件**文件夹（Supabase 的邮件很容易被识别为垃圾邮件）
2. 等待 1-2 分钟
3. 重新发送一次
4. 如果一直收不到，可能是你的邮箱服务商屏蔽了 Supabase 的发件域名，换一个邮箱试试（推荐 QQ 邮箱、网易邮箱、Gmail）

---

### Q4：页面打开是白屏

**可能原因**：

1. 程序还没完全启动好，等 10 秒再刷新
2. 浏览器地址栏访问的不是 `http://localhost:3000`
3. .env.local 配置错了，检查 URL 和 anon key 是否正确

**解决**：

- 看黑色窗口有没有报错信息
- 重启程序（关掉再重新双击 start.bat）

---

### Q5：提示"Project URL is required"或类似错误

**原因**：`.env.local` 文件配置错了。

**解决**：

1. 用记事本打开 `.env.local`
2. 检查：
   - 文件名是不是 `.env.local`（不是 `.env.local.txt`）
   - URL 是不是完整的（包含 `https://` 和 `.supabase.co`）
   - 等号两边有没有多余的空格
3. 修改保存后，重启程序

---

### Q6：执行 SQL 时报错 "already exists"

**原因**：之前已经执行过一次了。

**解决**：这是正常的，不用管。如果你**确定要重置数据库**（会清空所有数据）：

1. 在 Supabase 的 `Table Editor` 里手动删除所有表
2. 再重新执行一次 `init.sql`

> **警告**：删除表会丢失所有数据，请三思。

---

### Q7：分享给客户的报价单链接打不开

**原因**：你分享链接时，对方需要能访问你的本地程序。但本地程序只在你自己的电脑上运行，对方访问不到。

**说明**：当前版本是"本地运行"的，分享链接是 `/api/change-orders/[token]` 路径，需要你的程序正在运行才能访问。如果你想让客户直接打开：

- **方案 A**：你本地开着程序，截图发给客户看
- **方案 B**：把报价单内容复制粘贴到 Word/邮件发给客户
- **方案 C**：自己部署到 Vercel（需要一定技术能力，参考 `docs/architecture.md`）

---

### Q8：手机上能用吗？

**可以**。ScopeGuard 是移动端优先设计，手机浏览器访问 `http://localhost:3000` 也能用（前提是程序在你电脑上正在运行，且手机和电脑在同一个 WiFi 下）。

**手机访问电脑的步骤**：

1. 电脑上启动 ScopeGuard
2. 查看电脑的局域网 IP（在 cmd 里执行 `ipconfig`，找 IPv4 地址，形如 `192.168.1.100`）
3. 手机浏览器访问 `http://192.168.1.100:3000`

> 注意：需要关闭电脑的防火墙或者放行 3000 端口，否则手机访问不了。

---

### Q9：换电脑了怎么办？

1. 新电脑装 Node.js
2. 把整个 `scopeguard` 文件夹拷过去（或重新下载 ZIP）
3. 把 `.env.local` 文件也拷过去（或者重新创建）
4. 双击 `start.bat`
5. 用同一个邮箱登录，所有数据都在

> 数据存在 Supabase 云端，跟电脑无关。

---

### Q10：我不想用了，怎么删除所有数据？

1. 登录 Supabase：https://supabase.com
2. 进入你的项目
3. 左侧菜单 → `Table Editor`
4. 依次删除这 5 张表的数据（或删除整张表）：
   - `change_orders`
   - `scope_requests`
   - `deliverables`
   - `projects`
   - `profiles`
5. 如果想彻底删除：项目设置 → `General` → 最下方 `Delete project`

## 遇到问题怎么办

按以下顺序排查：

1. **重启程序**：关掉黑色窗口，重新双击 `start.bat`
2. **看窗口里的报错信息**：通常最后一行会写明原因
3. **查 FAQ**：上面 12 个问题覆盖了 90% 的情况
4. **求助懂技术的朋友**：把黑色窗口截图给他看
5. **去 GitHub 提 issue**：https://github.com/Ibuprovan/scopeguard/issues
